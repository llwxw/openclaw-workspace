## 核心结论

**OpenClaw是一个非线性系统，但我们一直在用线性工具设计它。**

这不是架构调整问题，是**设计范式的根本转换**：
- 旧范式：用乃氏图/根轨迹设计一个"可以通过调参达到最优"的静态系统
- 新范式：用描述函数/滞环/萨奇图/相平面解决非线性系统的稳定性问题

---

## 五个核心工具

### 1. 描述函数（Agent能力曲线）

```
K(n) = K_max · (1 - (n/n_max)^p)  当 n < n_max 时（线性衰减区）
K(n) = K_sat                         当 n ≥ n_max 时（饱和区）
```

**核心价值**：把"Agent在当前负载下的实际能力"建模为关于负载的函数，派发决策可以基于"这个任务在当前系统状态下能完成"的概率。

**触发机制**：
- 探索模式：n长期<0.3时注入探测脉冲，触碰饱和边界
- 保守模式：n>0.5时禁止探索，使用拟合参数下限

### 2. 滞环控制（消除高频颤震）

```
SPAWN_ON=45   # 高于此值派发
SPAWN_OFF=38   # 低于此值取消
```

**核心价值**：评分在38~45之间时保持当前状态不变，切断阈值附近小幅振荡时的高增益反馈通道（描述函数在a→0时F_r→∞的奇异性）。

**自适应调整**：
- n<0.3（低负载）→ 窄滞环（ON=42, OFF=38）
- n<0.5（中等负载）→ 标准滞环（ON=45, OFF=38）
- n>0.5（高负载）→ 宽滞环（ON=50, OFF=32）

### 3. 萨奇图无条件稳定性

```
n = queue_length / max_concurrent_tasks
条件：n < 0.5 → 无论时滞多大系统绝对稳定
```

**核心价值**：反直觉操作——越催促（提高派发速率）系统越不稳定；主动降低并发反而能稳定。

**操作**：检测到n>0.5时，立即降低max_concurrent_tasks使n回到0.4以下。

### 4. 相平面五态分类

```
H = 0.6×成功率 + 0.4×(1-归一化延迟)  # 健康度
L = queue_length / max_queue_length   # 负载度
```

| 区域 | 条件 | 控制动作 |
|------|------|---------|
| 稳定区 | H>0.7 且 L<0.5 | 常规运行，降低巡检频率 |
| 警戒区 | H<0.5 | 深度巡检，分析根因 |
| 过载区 | L>0.7 且 H>0.6 | 萨奇图降增益 |
| 发散区 | H<0.4 且 L>0.8 | 熔断，启动发射段模式 |
| **漂移态** | 各指标缓慢漂移无收敛发散 | 提高采样频率，跟踪变化率 |

### 5. 轨迹方向检测

```python
trajectory_vector = (H_now - H_3ago, L_now - L_3ago)
trajectory_direction = arctan2(vector[1], vector[0])
```

**核心价值**：系统在相平面上有流向，轨迹方向比位置更重要。
- 当前在稳定区但轨迹指向发散区 → 提前预警
- 当前在过载区但轨迹指向稳定区 → 减少干预，让系统自然恢复

## 三环架构

```
慢环（小时级）：SRE Agent → 相平面计算 → 轨迹分析 → 控制模式决策 → 更新CONTROL_META.yaml
中环（分钟级）：auto-spawn.sh → 读取控制模式 → 加载对应策略 → 实时派发决策
快环（实时）：派发执行与状态测量
```

三环通过配置文件解耦，各自独立运行，任何一环崩溃不影响其他环。

## 关键工程结论

### 最重要的发现（来自钱学森第10章）

继电器描述函数 `F_r = (4A·sinα)/(πa) · e^(-iθ)`

当输入振幅a→0时，等效增益F_r→∞。**这就是"阈值颤震"的根本物理原因**——不是代码bug，是物理定律。用线性调参无法解决这个问题，必须改变控制律结构（滞环）。

### 最反直觉的操作（来自第8章萨奇图）

不是"减小时滞"，而是"主动降增益"。当时滞增大时，正确的操作是减少并发数，而不是催促各个环节。这违反直觉但符合萨奇图的数学。

### 最被低估的章节（第12章火箭弹）

即使齐次方程的解全部衰减（系统本身稳定），驱动函数仍可能使输出发散。**"系统稳定吗"在变系数场景下是伪命题**。正确问法是：在当前输入条件下，系统能否满足性能准则？

### 最危险的状态（漂移态）

系统指标在多个维度上缓慢变化，无明显收敛或发散，但长期漂移出终点场时突然崩溃。这是"看起来正常但实际在恶化"的状态，在所有告警阈值触发之前悄悄发生。

## 一个悬而未决的根本问题

**OpenClaw到底是变系数线性系统还是非线性系统？**

钱学森明确指出这是两个不同类别。答案可能是：**两者都是**——在低负载区（线性区）表现为变系数线性系统，在高负载区（饱和区）表现为非线性系统。

这意味着单一一套分析框架是不够的，需要在不同运行区域切换分析范式。

**验证方法**：观察系统在低负载和高负载下的行为特征差异。如果两者有质的不同（低负载时参数变化率可预测，高负载时出现振荡和突变），就证明"双模式"诊断是正确的。
