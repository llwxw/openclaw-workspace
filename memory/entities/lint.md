# 🔍 Lint 工作流

> Wiki 健康检查流程

## 触发条件

| 场景 | 频率 |
|------|------|
| 每次修改后 | 死链检查 |
| 每周 | 全面检查 |
| 实体页数量逢 10 | 全面检查 |
| 重大变更后 | 手动 |

---

## 每周维护清单 (30分钟)

### 1. 死链检查 (5分钟)

```bash
grep -roh '\[\[[^]]*\]\]' memory/entities/*.md
# 验证每个链接对应文件存在
```

### 2. 矛盾扫描 (10分钟)

- 读取近期 Ingest 的页面
- 与旧页面对比是否冲突
- 标记: 

### 3. 过时检查 (5分钟)

- 检查 last_updated > 30天 的页面
- 判定是否仍有效
- 标记 stale 或更新

### 4. 孤儿检测 (5分钟)

- 无任何页面链接到的页面
- 添加相关链接或归档

### 5. 缺失检测 (5分钟)

- 提及但无页面的概念
- 建议创建

---

## 输出格式

```markdown
## 🔍 Lint 报告 (2026-04-11)

### 死链: 0 ✅
### 矛盾: 1 🟡
  - pageA vs pageB (待确认)
### 过时: 2 ⚠️
  - old-feature.md
### 孤儿: 1
  - unused.md
### 缺失建议: 3 💡
  - 建议创建 "xxx"
```
