# ⚠️ Edge Cases

> Wiki 处理不了的边界情况

---

## Wiki 擅长

| 场景 | 处理方式 |
|------|----------|
| 已经有记录的知识 | 直接读取 |
| 需要多来源综合 | 查询多个页面 |
| 需要沉淀新知识 | Ingest 流程 |

---

## Wiki 不擅长 / 边界情况

### 1. 新领域，无积累

```
表现：搜索不到相关内容
处理：
- 诚实说" wiki 里没有记录"
- 建议用户"是否现在开始记录？"
```

### 2. 过时信息

```
表现：找到的内容可能已失效
处理：
- 标记为可能过时
- 建议更新
```

### 3. 矛盾信息

```
表现：不同页面说法冲突
处理：
- 使用 [[contradictions]] 标记
- 保留双方，注明来源日期
```

### 4. 用户不想要

```
表现：用户说"不要记"
处理：
- 尊重用户选择
- 不写入
```

### 5. 敏感信息

```
表现：用户说"这个保密"
处理：
- 不写入 Wiki
- 如果已存在，删除
```

### 6. Wiki 太大

```
表现：搜索太慢，知识过载
处理：
- 考虑分 Wiki
- 用 tag 过滤
- 定期 lint 清理
```

### 7. 我不确定

```
表现：我不确定这个值对不对
处理：
- 说"不确定"
- 给出验证方法
- 不写入未确认的信息
```

---

## 错误处理原则

| 原则 | 说明 |
|------|------|
| 不知道就说不知道 | 不臆造 |
| 不确定就不写 | 避免错误沉淀 |
| 尊重用户 | 用户不要就不记 |
| 安全优先 | 敏感信息不存 |
