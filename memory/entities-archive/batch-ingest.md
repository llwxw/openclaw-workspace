# 📦 Batch Ingest

> 批量摄取多个 Source

## 场景

| 场景 | 适用 |
|------|------|
| 收集了 10+ 篇文章 | 批量 Ingest |
| 导入旧笔记 | 批量处理 |
| 项目启动 | 快速建立知识库 |

---

## 工作流

```
1. 准备 sources
   - 放入 raw/articles/ 或 raw/notes/
   - 命名格式: YYYY-MM-标题.md

2. 批量处理
   - 一次读取多个文件
   - 识别共性主题

3. 批量输出
   - 创建主索引页
   - 创建各 source 摘要页
   - 更新相关实体页 (可能 15-30 个)

4. 验证
   - 检查死链
   - 检查重复
```

---

## 批量 vs 单个

| 维度 | 单个 Ingest | 批量 Ingest |
|------|-------------|-------------|
| 数量 | 1 个 | 10+ 个 |
| 监督 | 高 (逐个确认) | 低 (自动化) |
| 质量 | 高 | 中 |
| 时间 | 长 | 短 |

---

## 建议

- **初期**: 少量多次，保持参与
- **熟悉后**: 可以批量，但定期回顾质量
- **任何时候**: 定期 lint 检查一致性
