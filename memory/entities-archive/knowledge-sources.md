# 📡 Knowledge Sources

> Wiki 的"食物"——知识从哪里来

## 输入渠道

| 渠道 | 类型 | 频率 | 示例 |
|------|------|------|------|
| **用户对话** | 交互 | 高 | 用户的提问、分享 |
| **网页** | 阅读 | 中 | Web Clipper 抓取 |
| **代码/文档** | 技术 | 中 | 读源码、API 文档 |
| **书籍** | 阅读 | 低 | 技术书籍 |
| **论文** | 研究 | 低 | 学术论文 |
| **播客/视频** | 视听 | 低 | 需转录 |

---

## 我(AI)的知识获取

| 来源 | 如何处理 |
|------|----------|
| 用户告诉我的 | 评估是否值得写入 wiki |
| 用户分享的链接 | 用 web_fetch 读取，写入 |
| 对话中发现的偏好 | 更新 preferences.md |
| 纠正/错误 | 写入 corrections.md |

---

## 知识判断

不是所有信息都值得写入 wiki：

| 值得写入 | 不需要写入 |
|----------|------------|
| 可复用的洞察 | 一次性答案 |
| 重要偏好/规则 | 日常闲聊 |
| 技能/工具学习 | 简单确认 |
| 错误/教训 | 过期信息 |

---

## 实际做法

```
用户发来链接 → 读取 → 问"要存到 wiki 吗？"
用户分享知识 → 评估 → 值得则写入
发现新技能 → 创建实体页 → 链接到 skills.md
```
