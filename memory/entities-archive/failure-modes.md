# 💥 Failure Modes

> 常见失败模式与避免

## Wiki 失败原因

| 失败模式 | 原因 | 解决方案 |
|----------|------|----------|
| **建了不用** | 热情消退 | 融入日常使用 |
| **过于复杂** | Schema 太严格 | 简化，从简单开始 |
| **质量下降** | 什么 都往里记 | 只记有价值的 |
| **维护真空** | 不 Lint | 定期维护 |
| **孤岛效应** | 页面不链接 | 强制链接 |

---

## 避免策略

### 1. 持续使用

```
- 每次重要对话后考虑写回
- 定期 lint 保持健康
- 周期性回顾
```

### 2. 质量控制

```
- 不是所有都要写
- 有洞见才写回
- 定期清理过时内容
```

### 3. 简单开始

```
- 先有一页
- 再慢慢扩展
- 不要追求完美Schema
```
